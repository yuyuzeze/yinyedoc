## 1. 外部インターフェース概要

本設計書は、新たなシステム（以下「本システム」）と格付判定システムとの間のWebAPI方式による外部インターフェースの設計方針である。

格付判定システムは、被保証者の保証料算定に使用する格付判定を行うシステムである。

保証料率算定に係る格付判定を行うために決算情報、保証審査結果、保証情報を送付し、保証料率算定に係る格付判定結果を受信する。

---

## 2. 連携方式設計

本システムと格付判定システムとの連携は、APサーバから格付判定システムのWebAPIをHTTPS POSTで呼び出すリクエスト／レスポンス方式（同期処理）を採用する。

## 3. 送受信データ設計

### 3.1 処理内容

本システム（林業業務システム）を起点として、格付判定サービスに対してREST形式のWebAPIを呼び出す。

格付判定サービスはFEG Azure基盤上のAzure App Serviceで提供される同期型WebAPIであり、本システムから財務情報等を送信すると格付結果を即時返却する。

通信はAzure Private Endpoint経由のみ許可されており、インターネット経由のアクセスは不可とする。

認証にはMicrosoft Entra IDのOAuth 2.0 Client Credentialsフローを使用し、本システムがEntra IDからアクセストークンを取得したうえで、Token及び判定材料をIFを通じて格付判定サービスに送信する。

格付判定サービスは判定を実施し、判定結果をIFを通じて本システムに返却する。

### 3.2 接続方式

格付判定サービスはAzure Private Endpoint経由の通信のみ許可。

本システムのJAFFIC Azure基盤から格付側のFEG Azure基盤への接続はMicrosoft Entra Private Access（Global Secure Access）を使用する。

### 3.4 トークン取得

> 【FEG v1.2対応】クライアントシークレット方式からManagedIdentity方式に変更

本システムが稼動するAzure VM に割り当てられた User-assigned Managed Identity を使用してトークンを取得する。HTTPエンドポイントには直接アクセスせず、Azure SDKの `ManagedIdentityCredential` を使用する。

■取得処理（VB.NET）

ManagedIdentityCredential を使用し、以下パラメータで取得する。

- `client_id`：User-assigned Managed Identity の Client ID
- `scope`：`api://<api-app-id>/.default`

■前提

- 林業業務システムが稼動するVMにUser-assigned Managed Identityを割当済み
- API側にて当該IdentityへApp Roleを付与済み

■補足

- `client_secret` は不要
- トークンはVM内部IMDS経由で自動取得

※ トークン取得通信はMicrosoft Entra Private Accessにより保護される

**表3-2 格付判定側から提供される情報**

> 【FEG v1.2対応】client-secret を削除

| tenant-id | 格付判定側Entra IDテナントID |
| client-id | User-assigned Managed Identity の Client ID |

## 4. エラー・例外処理設計

### 4.1 エラー処理設計

格付判定システムから返却されるHTTPステータスコードは以下の通り。

| 種別 | 内容 | 解決方針 |
|------|------|----------|
| 400 Bad Request | リクエスト形式不正 | 送信前バリデーション実施する。 |
| 401 Unauthorized | トークン未指定・無効 | 画面にエラーを提示する。トークンを自動リフレッシュして、失敗時は管理者アラートする。 |
| 403 Forbidden | 権限不足 | 画面にエラーを提示する。格付判定側にApp Role割当を依頼して、管理者アラートする。 |
| 404 Not Found | API不存在 | 画面にエラーを提示する。エンドポイントURL設定を確認して、管理者アラートする。 |
| 500 Internal Server Error | 格付判定側内部エラー | 画面にエラーを提示する。管理者アラートする。 |

### 4.2 例外処理設計

格付判定システムとの通信において発生する例外は、ログを記録して、画面にエラーを提示する。

| 種別 | 内容 | 解決方針 |
|------|------|----------|
| タイムアウト | API応答が規定時間を超過 | ログを記録して、画面にエラーを提示する。 |
| 接続エラー | ネットワーク障害・到達不能 | ログを記録して、画面にエラーを提示する。管理者アラートする。 |
