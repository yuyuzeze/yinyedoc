# 第9章 Excel出力方式

本章では、新たなシステム（以下「本システム」）における Excel ファイル出力の実装方式を定義する。  
現行システム（以下「旧システム」）で使用していた ExcelCreator v10 を v12 にアップグレードしたうえで継続使用し、既存の Excel テンプレートベースの帳票生成方式を .NET 10 環境に移行する。

| 項目 | 旧システム | 新システム（本章） |
|------|-----------|------------------|
| フレームワーク | .NET Framework 4.8 | .NET 10 |
| Excel ライブラリ | ExcelCreator v10 | ExcelCreator v12 |
| 出力方式 | テンプレート .xlsx ベース | テンプレート .xlsx ベース（継続） |
| テンプレート資産 | 既存 .xlsx テンプレート | 既存テンプレートを原則そのまま移行 |

---

## 9.1. ライブラリ選定理由

本システムにおいても ExcelCreator を継続採用する。その主な理由を以下に示す。

**表9-1　ライブラリ選定理由**

| 選定理由 | 詳細 |
|---------|------|
| 旧システムの継続性 | 旧システムで作成した Excel テンプレート資産（書式・罫線・レイアウト）をそのまま流用できる。テンプレートの作り直しコストを最小化できる。 |
| テンプレートベース帳票の品質 | 設計済みの Excel テンプレートにデータを流し込む方式のため、帳票レイアウトをデザイナー（Excel 操作者）が管理できる。コード変更なしでレイアウト修正が可能。 |
| .NET 10 対応 | ExcelCreator v12 は .NET 10 に対応している。旧バージョン（v10）は .NET Framework 専用のためバージョンアップが必要である。 |
| 日本語帳票への実績 | 日本の業務システム向けに長年使用されており、日本語フォント・日本語書式（和暦・金額等）への対応実績が豊富である。 |
| サポート・保守 | 国内ベンダー（アドバンスソフトウェア株式会社）による日本語サポートを受けられる。政府系システムの保守体制として信頼性が高い。 |

---

## 9.2. ExcelCreator v10 → v12 アップグレード

ExcelCreator v10 は .NET Framework 専用であり、.NET 10 では動作しない。旧システムで使用していた ExcelCreator v10 は .NET Framework 4.8 向けのバイナリであり、.NET 10 では直接参照・実行することができない。.NET 10 への移行に伴い、.NET 10 対応版である v12 へのアップグレードが必要となる。

**表9-2　移行時の主な変更点**

| 変更分類 | v10 の方式 | v12 の対応方法 |
|---------|-----------|--------------|
| 名前空間 | 旧名前空間 | v12 の名前空間に変更 |
| DLL 参照 | ExcelCreator.dll（旧） | v12 用 DLL に差し替え。.csproj の参照を更新する。 |
| ライセンス初期化 | v10 の方式 | v12 の初期化 API に変更。 |
| テンプレート | Open(path) | 変更なし。 |
| セル値設定 | SetValue(name, val) | 変更なし。 |
| リスト展開 | SetList(range, list) | v12 マニュアルに従い最新 API を適用する。 |
| 保存 | Save(stream) | 変更なし。 |

---

## 9.3. 実装方式

**表9-3　出力方式の基本方針**

| 項目 | 方針 |
|------|------|
| テンプレート方式 | 書式・罫線・タイトルを設定済みの .xlsx をテンプレートとして使用する。 |
| データ流し込み | 名前定義セルに SetValue でスカラー値（文字列、数値、日付等）を設定する。明細行は範囲名を指定して SetList で一括展開する。 |
| 出力先 | HTTP レスポンスとして MemoryStream 経由でブラウザに直接ダウンロードさせる。長期保存が必要な帳票のみ Azure Blob Storage（reports/）に追加保存する。 |
| テンプレート管理 | GovSystem.API/ExcelTemplates/ 配下にサブシステム別に格納する。 |

**表9-4　処理フロー**

| 順番 | 処理 | 内容 |
|-----|------|------|
| 1 | フロントエンド | Angular の「帳票出力」ボタン押下により、検索条件 DTO を body に含めた POST リクエストを送信する。レスポンスは blob 形式で受信する。 |
| 2 | データ取得 | Controller が DTO を受け取り、Service 経由で Repository → EF Core → Azure SQL からデータを取得する。 |
| 3 | テンプレート読込 | IExcelExportService.ExportAsync にテンプレート名と fillAction を渡す。ExcelCreator v12 がテンプレート .xlsx を読み込む。 |
| 4 | データ設定 | fillAction 内で SetValue（固定セルへのスカラー値設定）および SetList（明細行の範囲名一括展開）を実行する。 |
| 5 | バイト列返却 | MemoryStream に Save し、byte[] として Controller に返却する。 |
| 6 | レスポンス | Controller が File レスポンスを返す。日本語ファイル名は Uri.EscapeDataString() でエンコードする。ブラウザが .xlsx としてダウンロードする。 |
