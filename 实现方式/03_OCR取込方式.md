# 第3章 OCR取込方式

---

## 3.1 OCR取込の目的

従来は紙帳票や手書き書類の内容を、担当者が手入力する作業が存在していた。  
本システムではOCR（光学文字認識）により、これらの情報を自動的にデータ化し、入力作業の効率化および業務負担の軽減を図ることを目的とする。

---

## 3.2 機能概要

本機能は、本システムに導入するOCR製品で出力する各種書類のCSVファイルを業務システムに取り込み、入力画面にOCRファイルを選択して項目値を自動入力する機能である。

## 3.4 処理内容

### 表3-1 OCR処理内容

| ステップ | 実行者 | 操作・処理内容 |
|---|---|---|
| 1. 帳票スキャン＆OCR | OCRオペレータ | 帳票をスキャンし、AnyForm OCRで読取・Verifier確認を実施する。 |
| 2. CSV出力＆バッチ起動 | AnyForm | スケジューラーで定期的に出力ファイルのバッチを呼び出す。 |
| 3. BlobへAzCopyアップロード | AnyForm | バッチでBlob Storageへアップロードする。 |
| 4. 業務画面にBlob一覧確認 | 業務担当者 | 業務システムの「OCR取込」画面でBlob上のCSVファイル一覧を表示する。 |
| 5. 取込対象を選択 | 業務担当者 | ファイル名・日時・帳票種別を確認し、取り込むCSVを選択する。 |
| 6. CSVファイルの反映 | システム | 選択したCSVをマッピング処理して、画面に反映する。（※画面の項目自動チェック無し） |
| 7. 内容確認＆申請 | 業務担当者 | 画面に反映した内容を確認して、申請を行う。 |
| 8. 伝票登録 | システム | DBへ登録する。 |
| 9. Blobファイルの世代管理 | システム管理者 | 基盤側のBlob Storageのファイル管理方針を踏襲する。 |

## 3.6 バッチ設計

**バッチ処理内容：** AzCopyの機能を用いてAnyFormから出力したCSVファイルをBlob StorageへアップロードするAnyFormは1テンプレートにつき1種類のCSVを出力する仕組みであり、帳票の種類ごとにテンプレートと対応するCSV形式が固定される。

**利用の前提：** 実行ライブラリー（azcopy.exe）を端末に配置。特定のBlob StorageのパスにAzCopyでアップロードするための共有アクセス署名（SAS）申請、端末のスケジューラー（タスクスケジューラ）設置は必要である。

**バッチのコンフィグファイル項目：** 中間フォルダパス、ログファイルパス、署名（SAS）文字列となる。

### 処理の流れ

1. AnyFormの設定にしたがってCSVファイルを中間フォルダに出力する。
2. スケジューラーで定期的にバッチを呼び出す。
3. バッチで中間フォルダのファイルをBlob StorageへアップロードするアップロードしたファイルはBlobの処理済みフォルダへ移動する。アップロード失敗のファイルを中間フォルダに放置する。
4. 次回にバッチを実行する際に、今回アップロードのファイルと以前に失敗したファイルを共にアップロードを実行する。

### 3.7.3 入力画面反映

#### 3.7.3.1 選択したOCRファイルの読込

一覧画面のファイルを選択して、取込ボタンを押下して、全部行のCSV項目値を取込する。

#### 3.7.3.2 項目のマッピング

項目のマッピングテーブルを利用して、OCRファイルの項目を画面の項目に反映する。

**表3-1 マッピングテーブル設計**

| 項目 | 型 | 説明 |
|---|---|---|
| 画面ID | nvarchar | 画面特定用ID |
| 画面項目ID | nvarchar | 画面項目識別用ID |
| 帳票ID | nvarchar | 帳票特定用ID |
| 帳票項目ID | nvarchar | CSV項目物理名 |
| 変換要否 | bit | 0：不要　1：必要 |
| 変換ID | int | 変換テーブルのキー |
| チェック種類 | int | 例：1：長さチェック　2：数字チェック　3：日付形式チェック |

#### 3.7.3.3 個別変換

変換要否が「1：必要」の項目は個別変換を行う。

**表3-2 個別変換テーブル設計**

| 項目 | 型 | 説明 |
|---|---|---|
| 変換ID | int | |
| 変換種類 | int | 例：1：日付変換　2：保証番号 |
| 変換フォーマット | nvarchar | 変換のフォーマット　例：yyyy/MM/dd |
| 変換元 | nvarchar | 変換元　例：普、根、当 |
| 変換先 | nvarchar | 変換先　例：フ、ネ、ト |

#### 3.7.3.4 反映チェック

チェック種類が設定される項目値をチェックする。  
チェック種類により、長さチェック、数字チェック、日付形式チェックなどを行う。

**入力画面反映の流れ（図3-4）**

```
選択したOCR          項目の            個別変換          反映チェック        画面反映
ファイルの読込  →   マッピング  →  （変換要否=必要）  →              →
                                  ↑                      ↑
                          個別変換不要の場合          個別変換と反映チェック不要の場合
                          （スキップ）                （スキップ）
```

## 3.8 画面構成

### 3.8.1 ファイル一覧

- Blobフォルダ構成を取得してツリー表示し、フォルダをクリックすると開閉可能である。
- 更新ボタン押下時、ツリーを最新化する。
- 検索テキストに検索文字入力時、ファイルを洗い出して、検索結果を表示する。
- ファイルを選択後、取込ボタン押下時、OCRファイルの内容によって、親画面の項目に値を設定する。
- ファイルを選択後、プレビューボタン押下時、プレビューダイアログが表示される。

### 3.8.2 プレビュー画面

- CSVファイルの内容はテーブルの形式で表示される。
- テーブルの行を選択して、取込ボタン押下時、選択した行の内容によって、親画面の項目に値を設定する。

## 3.9 CSVファイル項目と画面項目のマッピング

DBに機能IDごとにCSV項目、画面項目を事前に登録する。  
登録したCSV項目をキーとして、項目値で画面の該当する項目に設定する。
