# 第2章 ソフトウェア一覧

本システムでは、保守性、開発効率、および Azure 環境との親和性を重視し、以下の技術・ソフトウェアを選定する。

---

## 2.1. フロントエンド（SPA）側では：Angular + TypeScript + Angular Material

Angular を採用する理由は、画面開発に必要な機能を標準的に備えており、設計および実装方式の統一を図りやすいためである。

また、TypeScript を前提とした開発により、型安全性を確保しやすく、実行時不具合の抑制に寄与する。

さらに、画面遷移、HTTP クライアント、フォーム管理等を一貫した方式で実装できるため、担当者間の引継ぎや長期保守の観点でも適している。

複雑な入力要件に対しても、ReactiveFormsModule を用いることで、フォーム状態およびバリデーションを構造的に管理しやすい。

Angular 20 は LTS（Long Term Support）バージョンであり、長期間にわたりセキュリティ更新およびサポートが提供される。

## 2.2. バックエンド（REST API）：.NET Core + Entity Framework Core

.NET Core は Microsoft 提供のアプリケーション開発・実行基盤であり、認証設定、パフォーマンス監視、および Azure 環境へのデプロイメントとの親和性が高い。

これにより、インフラ構築および運用保守の負担軽減が期待できる。

また、.NET Core は長期間にわたりセキュリティ更新およびサポートが提供される。

Entity Framework Core の ORM 機能を活用し、既存のデータベース定義（テーブル定義）に基づくエンティティクラスの生成や、オブジェクト指向に基づくデータアクセスを実現する。

これにより、CRUD 操作を安全かつ可読性高く実装できる。

Azure SDK および Microsoft.Identity.Web 等の Microsoft 提供パッケージも .NET 10 のサポートライフサイクルに準拠する。

Serilog、AutoMapper、FluentValidation、Polly 等の OSS パッケージについては、NuGet での利用実績が広く、継続的にメンテナンスされているライブラリを採用している。

商用ライブラリ（ExcelCreator）については、有償保守契約によりサポート及びセキュリティパッチの提供が維持される。
